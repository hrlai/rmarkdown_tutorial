Diamond sizes

2016-08-25

library(ggplot2)
library(dplyr)
smaller <- diamonds %>% 
  filter(carat <= 2.5)

We have data about 53,940 diamonds. Only 126 (0.998%) are larger than 2.5 carats. The distribution of the remainder is shown below:

smaller %>% 
  ggplot(aes(carat)) + 
  geom_vline(xintercept = seq(0, 2.5, 0.1), colour = "lightgrey") +
  geom_freqpoly(binwidth = 0.01) +
  theme_classic()

Most carat ratings cluster around multiplications of 0.1 (grey vertical lines).

Dependent chunks and caching

Yihui Xie explains stuff much better than R4DS, which can be sloppy at times

Without caching

dat = matrix(runif(100000), ncol=5)
dat[, 3] = -.2 * dat[, 1] + .8 * dat[, 2] # to make the plot less boring
pairs(dat)

Without caching, we spent 17.4 seconds.

With caching

dat2 = matrix(runif(100000), ncol=5)
dat2[, 3] = -.2 * dat2[, 1] + .8 * dat[, 2] # to make the plot less boring
pairs(dat2)

With caching, we spent 0.01 seconds (on subsequent knit without making changes to the depended chunk(s)).
